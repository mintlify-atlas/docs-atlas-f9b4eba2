drei (German for "three") is a growing collection of useful helpers and fully functional, ready-made abstractions for [@react-three/fiber](https://github.com/pmndrs/react-three-fiber). Whether you need cameras, controls, shaders, loaders, or performance utilities, drei has you covered — so you can focus on building your 3D experience rather than reinventing primitives.

## Quick start

<Steps>
  <Step title="Install drei">
    Add drei and its peer dependencies to your project:

    <CodeGroup>
    ```bash npm
    npm install @react-three/drei @react-three/fiber three
    ```
    ```bash yarn
    yarn add @react-three/drei @react-three/fiber three
    ```
    ```bash pnpm
    pnpm add @react-three/drei @react-three/fiber three
    ```
    </CodeGroup>
  </Step>
  <Step title="Import what you need">
    drei exports everything from a single package entry point:

    ```jsx
    import { OrbitControls, Environment, PerspectiveCamera } from '@react-three/drei'
    ```
  </Step>
  <Step title="Use inside a Canvas">
    All drei components are designed to be used inside a `@react-three/fiber` Canvas:

    ```jsx
    import { Canvas } from '@react-three/fiber'
    import { OrbitControls, Environment, Box } from '@react-three/drei'

    export default function App() {
      return (
        <Canvas>
          <PerspectiveCamera makeDefault position={[0, 0, 5]} />
          <Environment preset="city" />
          <Box>
            <meshStandardMaterial />
          </Box>
          <OrbitControls />
        </Canvas>
      )
    }
    ```
  </Step>
</Steps>

<Note>
  drei uses the standalone [`three-stdlib`](https://github.com/pmndrs/three-stdlib) package instead of `three/examples/jsm`, giving you better tree-shaking and more stable imports.
</Note>
